# AI Command Fixer MVP 封版人工验收清单

适用范围：MVP Web 工具页封版前人工验收。自动化基线仍以 `npm test` 为准；长命令细节回归用例见 `docs/testing.md`。

## 验收前准备

- [ ] 在项目根目录运行 `npm test`，确认自动化测试全部通过。
- [ ] 运行 `npm start` 启动本地静态服务。
- [ ] 打开 `http://127.0.0.1:4173`，使用普通桌面浏览器完成主流程验收。
- [ ] 准备一个移动端窄屏视口，或使用浏览器响应式模式完成移动端基础验收。

## 页面加载与默认状态

- [ ] 页面可以正常加载，无空白页、无明显样式错乱、无控制台致命错误。
- [ ] 首屏是 Web 工具页本体：输入区、修复操作、结果区都可见或可通过轻微滚动到达。
- [ ] 初始结果区为空状态，不显示历史命令内容。
- [ ] 初始状态不主动展示多命令拆分结果，也不提示必须拆成多条命令。
- [ ] 刷新页面后，输入框和结果区不会恢复上一轮粘贴过的命令。

## 默认单命令模式

- [ ] 粘贴一段包含说明文字和一条长命令的内容，点击“修复命令”后默认输出 1 条命令。
- [ ] 粘贴一条包含 `&&`、管道、引号、括号或 Python `-c` 的长命令，默认仍按单条命令修复。
- [ ] 默认模式不会因为命令内部换行就主动拆成多条结果。

## 复杂 Shell Block：for/do/done

- [ ] 输入包含 `for ... do ... done` 的复杂 shell block，点击“修复命令”后整体保持为 1 条可复制命令。
- [ ] `for`、`do`、`done` 结构不被误删、不被错误合并成无法识别的 token。
- [ ] block 内部的路径、变量、引号和分号保持原有语义。

## SQL FROM 空格

- [ ] 输入 SQL 字符串中 `FROM` 后发生真实换行的命令，例如 `FROM` 换行到 `geo_raw_responses`，修复后为 `FROM geo_raw_responses`。
- [ ] 输入 `FROM` 换行到 `geo_dashboard_summary`，修复后为 `FROM geo_dashboard_summary`。
- [ ] 验证不会出现 `FROMgeo_raw_responses`、`FROMgeo_dashboard_summary` 这类 SQL 关键字边界误合并。
- [ ] SQL 中普通可读性空格不会被过度删除。

## 日期断点

- [ ] 输入 `2026-` 换行到 `06-06`，修复后为 `2026-06-06`。
- [ ] 输入 `2026-06-` 换行到 `12`，修复后为 `2026-06-12`。
- [ ] 日期断点修复不会引入额外空格，例如不会变成 `2026-06- 12`。

## 字段名断点

- [ ] 输入 Python 字符串或列表中的 `citation_l` 换行到 `ist`，修复后为 `citation_list`。
- [ ] 输入 SQL 标识符 `platform_` 换行到 `name`，修复后为 `platform_name`。
- [ ] 输入 SQL 标识符 `r.` 换行到 `question_id`，修复后为 `r.question_id`。
- [ ] 输入 `recommended_` 换行到 `questions`，修复后为 `recommended_questions`。
- [ ] 字段名断点修复不会删除字典、SQL、Python 代码里的正常语义空格。

## 路径、目录名与文件名断点

- [ ] 输入路径目录名 `raw_` 换行到 `exports`，修复后为 `raw_exports`。
- [ ] 输入长文件名 `jingdong_` 换行到日期或后续文件名片段，修复后保持为连续文件名。
- [ ] 输入表名或路径片段 `geo_` 换行到 `raw_responses`，修复后为 `geo_raw_responses`。
- [ ] 修复后路径仍保留原有引号、斜杠结构和文件扩展名。

## 复制能力

- [ ] 每条结果卡片都有单条“复制”按钮。
- [ ] 点击单条“复制”按钮后，剪贴板内容等于当前结果卡片里的修复命令。
- [ ] 复制成功后页面有明确反馈，例如显示“已复制命令 X”。
- [ ] 支持结果默认以可编辑文本框展示，点击文本框只定位光标或选择文本，不触发复制。
- [ ] 编辑结果后，单条“复制”按钮复制编辑后的当前内容。
- [ ] 如果出现多条结果，只在多条结果场景显示“复制全部”，且内容按换行分隔。

## 本地暂存

- [ ] 默认修复命令后，不会自动新增本地暂存项。
- [ ] 未点击“暂存”时刷新页面，输入框、结果区和本地暂存区都不会恢复上一轮命令内容。
- [ ] 点击单条结果旁的“暂存”后，本地暂存区出现该条修复后的完整命令和自动短标题。
- [ ] 暂存项只保存修复后的完整命令、可编辑标题和必要元数据：`id`、`title`、`command`、`createdAt`、`expiresAt`。
- [ ] 暂存标题可直接编辑，修改后有保存反馈，刷新后仍保留。
- [ ] 旧版无 `title` 的暂存项可读取，并会在当前浏览器本地补默认标题。
- [ ] localStorage key 使用 `ai-command-fixer.localStash.v1`，数据结构不包含原始输入、修复前文本、修复点详情、风险详情、命令 hash、路径、URL、来源页面或统计字段。
- [ ] 暂存成功提示清楚说明暂存内容仅保存在当前浏览器本地，并会按过期时间自动清理。
- [ ] 可选择 `1 小时`、`24 小时`、`7 天` 过期时间，默认是 `24 小时`。
- [ ] 刷新页面后，未过期暂存项仍可见。
- [ ] 页面加载、打开本地暂存区或新增暂存时，已过期暂存项会自动清理。
- [ ] 暂存满 20 条后，第 21 条新增会被阻止，并提示删除旧暂存项或清空全部；不会自动覆盖旧项。
- [ ] 本地暂存区支持复制单条，复制内容等于该暂存项的修复后完整命令。
- [ ] 本地暂存区支持删除单条。
- [ ] 本地暂存区支持清空全部。

## 隐私与本地存储

- [ ] 断网或禁用网络后，已加载页面仍可完成修复和复制核心流程。
- [ ] 浏览器开发者工具 Network 中，粘贴命令和点击修复时没有向外部服务上传命令内容。
- [ ] 刷新页面后，上一轮命令内容不会出现在输入框、结果区或历史列表中。
- [ ] Local Storage、Session Storage 和 IndexedDB 中不保存自动命令历史；主动暂存除外。
- [ ] 主动暂存之外，允许保存的仅限非敏感偏好设置，例如显示选项或自定义前缀。
- [ ] FAQ、README、部署说明、统计规范中的隐私口径一致：默认不保存命令，主动暂存只保存当前浏览器本地的修复后完整命令和可编辑标题，并按过期时间清理。
- [ ] 统计事件不得包含原始输入、修复后命令、暂存命令、命令 hash、精确长度、路径片段、自定义前缀文本或剪贴板内容。

## 移动端基础可用

- [ ] 在约 375px 宽度下页面为单列布局，无横向滚动导致的核心功能不可用。
- [ ] 输入区、修复按钮、清空或示例操作、结果区都可触达并可点击。
- [ ] 长命令结果可阅读、可滚动或可选择，不会溢出遮挡复制按钮。
- [ ] 单条复制按钮、“复制全部”和默认可编辑结果框在移动端可用。
- [ ] 本地暂存展开区、复制、删除和清空操作在移动端可用。
- [ ] 设置或辅助选项在移动端不会遮挡主流程。

- [ ] `npm test` 自动化基线通过。
