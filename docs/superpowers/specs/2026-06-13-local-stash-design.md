# AI Command Fixer 本地暂存功能设计

日期：2026-06-13

## 1. 背景

用户在使用 AI Command Fixer 修复长命令后，经常会在部署、排查、SQL 验证或数据导出流程中反复使用同一条修复结果。当前工具默认不保存命令历史，这符合隐私优先原则，但也导致用户需要反复粘贴、修复和复制。

本功能用于解决“临时保存修复后的命令，稍后继续复制使用”的场景。它不引入服务器，不做云同步，不保存原始输入，也不把暂存功能包装成历史记录。

## 2. 产品决策

- 将“本地暂存”加入 MVP，但保持克制。
- 默认不自动保存任何命令。
- 只有用户主动点击“暂存”时，才保存修复后的完整命令。
- 暂存内容只保存在当前浏览器本地。
- 暂存项必须有过期时间，并在页面加载或用户操作时自动清理。
- 暂存不是历史记录，不做搜索、标签、收藏夹或云同步。
- 隐私文案需要从“完全不保存命令”调整为“默认不保存；主动暂存除外”。

## 3. 目标

- 让用户可以临时保存修复后的完整命令，稍后继续复制。
- 不需要后端服务器、账号、数据库或 API。
- 不保存原始输入、AI 回复全文、修复前片段或命令历史。
- 让用户清楚知道暂存内容只保存在当前浏览器本地。
- 支持自动过期、单条删除和清空全部。

## 4. 非目标

- 不自动提取 SQL。
- 不只保存 SQL 片段。
- 不做命令历史记录。
- 不做云同步或跨设备同步。
- 不做分享链接。
- 不做搜索、标签、收藏夹。
- 不做加密保险箱或密码保护。
- 不把暂存内容发送到统计、日志或错误监控。

## 5. 功能范围

### 5.1 暂存对象

保存对象为“修复后的完整命令文本”。

不保存：

- 原始输入。
- 原始 AI 回复。
- 修复前文本。
- 自动修复点详情。
- 风险提示详情。
- 自定义前缀文本以外的敏感输入。

### 5.2 暂存入口

在每条修复结果附近提供“暂存”入口。

推荐入口：

- 与单条“复制”按钮并列。
- 按钮文案：`暂存`。
- 暂存成功后状态提示：`已暂存，24 小时后自动清理。`

### 5.3 过期时间

MVP 支持三个过期选项：

- `1 小时`
- `24 小时`
- `7 天`

默认：`24 小时`。

用户可以在暂存按钮附近或本地暂存区中选择过期时间。若 UI 需要简化，第一版可在设置区提供统一默认过期时间。

### 5.4 暂存上限

- 最多保存 20 条。
- 超过上限时，提示用户删除旧暂存项或清空全部。
- 不自动覆盖旧项，避免用户误以为内容还在。

### 5.5 暂存区

页面下方增加“本地暂存”折叠区。

每条暂存项显示：

- 创建时间。
- 过期时间。
- 命令预览。
- 复制按钮。
- 删除按钮。

暂存区提供：

- 清空全部。
- 过期项自动清理说明。
- 隐私提示：`暂存内容仅保存在当前浏览器本地。`

### 5.6 自动清理

以下时机清理过期项：

- 页面加载时。
- 新增暂存时。
- 打开暂存区时。
- 用户点击清理或清空操作时。

## 6. 数据结构建议

存储位置：浏览器 `localStorage`。

建议 key：

```text
ai-command-fixer.localStash.v1
```

建议结构：

```json
{
  "version": 1,
  "items": [
    {
      "id": "local-generated-id",
      "command": "fixed command text",
      "createdAt": 1780000000000,
      "expiresAt": 1780086400000
    }
  ]
}
```

说明：

- `id` 只需本地唯一。
- 不保存原始命令。
- 不保存来源页面、URL、路径分类、token 片段或命令 hash。
- 不把 `command` 传给任何统计、日志或 URL。

## 7. 隐私文案调整

原隐私承诺需要调整为：

> 默认不保存任何命令。只有当你主动点击“暂存”时，修复后的命令才会保存在当前浏览器本地，并按过期时间自动清理。

FAQ、README、部署文档、统计规范需要同步：

- 不上传命令。
- 不保存命令历史。
- 主动暂存是用户显式行为。
- 暂存内容仅在当前浏览器本地。
- 可以单条删除或清空全部。

## 8. 验收标准

- 默认修复命令后，不会自动出现在本地暂存区。
- 点击“暂存”后，修复后的完整命令出现在本地暂存区。
- 暂存项不包含原始输入。
- 刷新页面后，未过期暂存项仍可见。
- 过期项在页面加载或打开暂存区时自动清理。
- 支持复制暂存项。
- 支持删除单条暂存项。
- 支持清空全部暂存项。
- 超过 20 条时阻止新增并提示用户清理。
- localStorage 中不保存命令历史列表之外的原始输入或修复前文本。
- 统计事件不得包含暂存命令内容、命令 hash、精确长度或路径片段。

## 9. 交互设计补充

### 9.1 设计原则

本地暂存应该像“临时便签”，而不是历史记录、收藏夹或命令管理器。

- 主修复路径仍然是：粘贴输入 -> 修复 -> 核对 -> 复制。
- 暂存入口只在用户看到修复结果后出现，不在输入区提前强调。
- 不使用“历史”“记录”“保存全部”“最近使用”等容易误导的文案。
- 所有暂存相关位置都要重复说明：`仅保存在当前浏览器本地`、`按过期时间自动清理`。
- 默认折叠，不占用首屏主任务空间。
- 不展示搜索、标签、星标、分类、批量导出等二级能力。

### 9.2 暂存按钮位置

暂存按钮放在每条修复结果的操作栏中，与复制按钮同组，但视觉权重低于复制按钮。

推荐层级：

1. 主要按钮：`复制`
2. 次要按钮：`暂存`
3. 结果框点击复制仍然保留

布局建议：

- 桌面端：修复结果卡片右上角或结果标题行右侧，顺序为 `复制`、`暂存`。
- 移动端：结果下方操作栏，`复制` 占主要宽度，`暂存` 使用次要按钮。
- 暂存按钮不放在全局操作区，避免用户误以为会暂存输入区内容或全部结果。
- 暂存按钮不放在复制按钮之前，避免打扰“立刻复制执行”的主流程。

按钮状态文案：

| 状态 | 按钮/反馈文案 |
| --- | --- |
| 默认 | `暂存` |
| 暂存中 | `暂存中...` |
| 暂存成功 | `已暂存，24 小时后自动清理。` |
| 已暂存同一条 | `已在本地暂存区` |
| 暂存失败 | `暂存失败，请重试。` |
| 达到上限 | `本地暂存已满，请删除旧项后再暂存。` |

同一条命令重复点击暂存时，MVP 不新增重复项，直接提示 `已在本地暂存区`。是否为同一条命令只在本地判断，不生成或上报命令 hash。

### 9.3 过期时间选择

过期时间选择应靠近暂存动作，但不能让每次复制前多一步决策。

推荐方案：结果区提供轻量下拉选择，默认 `24 小时`。

- 在 `暂存` 按钮左侧或按钮组内显示 `保留 24 小时` 下拉。
- 下拉选项为：`1 小时`、`24 小时`、`7 天`。
- 当前选择只影响下一次暂存，不改变已经暂存的项目。
- 选择变更可以作为本地偏好保存，但不保存任何命令内容。

简化备选方案：如果首版界面空间不足，可以把过期时间放在本地暂存区顶部：

- 顶部文案：`新暂存默认保留：24 小时`
- 用户可通过下拉切换默认值。
- 暂存成功反馈必须带具体过期时间，例如 `已暂存，7 天后自动清理。`

不建议：

- 不使用日期时间选择器，MVP 只需要固定选项。
- 不提供“永久保存”。
- 不把过期时间隐藏在设置区深处，否则用户暂存时无法建立隐私预期。

### 9.4 本地暂存区位置与结构

本地暂存区位于主结果区下方、设置区上方，默认折叠。

标题建议：

```text
本地暂存
```

标题旁辅助文案：

```text
仅保存在当前浏览器本地，过期后自动清理
```

折叠区头部信息：

- 左侧：`本地暂存`
- 计数：`3/20`
- 最近清理反馈：`已清理 2 条过期暂存`
- 右侧操作：`展开` / `收起`

展开后顶部操作：

- `清理过期项`
- `清空全部`
- 隐私提示：`暂存内容不会上传，也不会跨浏览器同步。`

每条暂存项展示：

- 第一行：创建时间与过期倒计时，例如 `今天 14:32 暂存 · 23 小时后清理`
- 第二行：命令预览，最多 2-3 行，超出省略
- 操作：`复制`、`删除`

暂存项不展示来源、原始输入、修复说明、风险详情或自动修复点，避免把它变成完整历史记录。

### 9.5 状态文案

空状态：

```text
还没有本地暂存
主动点击修复结果旁的“暂存”后，命令才会保存在当前浏览器本地。
```

有内容状态提示：

```text
这些命令仅保存在当前浏览器本地，并会按过期时间自动清理。
```

满额状态：

```text
本地暂存已满（20/20）
请删除不需要的暂存项，或清空全部后再继续暂存。
```

过期自动清理反馈：

```text
已清理 2 条过期暂存。
```

无过期项反馈：

```text
没有需要清理的过期暂存。
```

删除单条确认：

```text
删除这条本地暂存？
```

删除成功：

```text
已删除这条本地暂存。
```

清空全部确认：

```text
清空全部本地暂存？此操作只会删除当前浏览器里的暂存命令。
```

清空成功：

```text
已清空全部本地暂存。
```

复制暂存项成功：

```text
已复制暂存命令。
```

localStorage 不可用：

```text
当前浏览器无法使用本地暂存，但仍可复制修复结果。
```

### 9.6 删除与清空交互

删除单条：

- 删除按钮放在暂存项右侧或底部操作区，视觉权重低于复制按钮。
- 删除前建议二次确认，避免误删临时命令。
- 删除后从列表中移除，并显示轻提示。

清空全部：

- 只在暂存区展开后显示。
- 使用危险操作样式，但不要做成主按钮。
- 必须二次确认。
- 确认文案明确“当前浏览器”，避免用户误解为云端清理。

过期清理：

- 页面加载、打开暂存区、新增暂存时自动清理，不需要打断用户。
- 如果清理发生在用户可见操作后，显示非阻塞轻提示。
- 不弹窗提示过期清理，避免打扰主流程。

### 9.7 可用性与视觉层级

- `复制` 是主操作，`暂存` 是辅助操作。
- 本地暂存区默认折叠，标题可见但内容不抢占修复结果空间。
- 暂存项用紧凑列表，不使用大卡片墙，避免产生“命令库”心智。
- 命令预览使用等宽字体，最多展示 2-3 行。
- 所有危险或破坏性动作使用明确文案，不只依赖图标。
- 暂存区内可以使用小型图标辅助复制、删除、展开，但关键文案必须可读。

### 9.8 面向前端实现的验收补充

- 暂存按钮只保存对应修复结果，不保存输入框内容。
- 同一条修复结果重复暂存时不新增第二条。
- 过期时间选择改变后，新暂存使用新值，旧暂存的 `expiresAt` 不被改写。
- 暂存区计数只统计未过期项目。
- 自动清理过期项后，计数同步更新。
- 暂存区空状态不出现“历史记录”相关文案。
- 清空全部只清除本地暂存 key，不清除用户偏好和自定义命令前缀。
- localStorage 不可用时，暂存入口给出降级提示，不影响复制。

## 10. Agent 分工

- 设计 agent：确认“暂存”按钮和“本地暂存”折叠区的信息层级，避免打扰主流程。
- 前端 agent：实现本地暂存、过期清理、复制、删除、清空和上限。
- 测试 agent：补自动化测试和人工验收清单，重点覆盖默认不保存、主动暂存、过期清理和隐私边界。
- 产品经理：审核文案、验收标准和 agent 结果。

## 11. 后续候选

- 暂存项重命名。
- 手动导出 `.txt` / `.sh`。
- 从 Python `-c` 中提取 SQL 片段。
- CLI 或 Codex skill 中的批量修复输出。

这些不进入当前 MVP 暂存功能。
