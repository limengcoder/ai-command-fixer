更新：2026-06-12

## 1. 实现原则

- 第一版采用零依赖静态 Web：`index.html` + `src/*.js` + `styles.css`。
- 不引入后端、数据库、账号系统或云端 API。
- 核心解析逻辑与 UI 分离，解析引擎可以用 Node 内置测试单独验证。
- 默认不保存命令历史；只用 `localStorage` 保存偏好和自定义命令前缀。
- 首屏直接是工具，不做营销落地页。
- 先保证真实命令修复准确，再做视觉润色。
- MVP 默认采用单命令模式，优先服务 AI 生成长命令修复。
- 多命令识别仅保留为显式高级能力，不作为默认 UI 流程。
- skill、CLI、浏览器插件和批量处理暂不进入第一版实现。

## 3. 工作包

### 3.1 解析引擎

负责人：前端 Agent，主线程验收。

文件：`src/command-parser.js`、`tests/command-parser.test.js`

实现内容：

- `parseCommands(input, options)`：从整段文本中输出命令结果列表。
- 默认 `parseCommands(input)` 按单条长命令处理。
- 显式 `splitMode: "auto"` 时，才启用多命令智能拆分。
- 文本预处理：
  - 统一换行。
  - 去掉 Markdown 代码块边界。
  - 去掉常见终端提示符。
  - 忽略说明文字行。
- 候选命令识别：
  - 常见命令前缀。
  - 绝对路径可执行文件。
  - 带命令前缀的多行参数。
- 状态机合并：
  - 反斜杠续行。
  - 未闭合引号。
  - 未闭合括号、方括号、花括号。
  - 缩进行并入当前命令。
  - 默认模式不因下一行看起来像新命令而自动切分。
  - 显式 auto 模式才根据新命令起点切分。
  - `for/do/done`、`if/then/fi`、`while/do/done` 等 shell block 保持为同一条命令。
- 空白归一：
  - 将换行和缩进造成的连续空白压成单个空格。
  - 不删除普通参数间空格。
- 提示：
  - 记录修复次数和说明。
  - 检测 `<<EOF` 并标记为不支持。
  - 检测 `rm -rf`、`DROP DATABASE` 等高风险片段。

验收：

- 设计文档第 9 节样例测试全部通过。
- 示例 Python `-c` SQL 折行能正确修复为单行。
- 用户反馈的 `for d in ...; do ...; done` 长命令只输出 1 条命令。
- `/home/venvs/geo/bin/ ⏎ python` 修复为 `/home/venvs/geo/bin/python`。

### 3.2 UI 与交互

负责人：设计 Agent 给建议，前端 Agent 实现，主线程验收。

文件：`index.html`、`styles.css`、`src/app.js`

实现内容：

- 顶部说明：产品名、本地处理隐私提示，文案强调“本地处理”“不保存命令历史”“离线可用”。
- 输入区：大文本框，支持粘贴整段 AI 回复。
- 操作区：
  - 修复。
  - 清空。
  - 加载示例。
  - 设置开关。
- 结果区：
  - 默认显示一条修复结果。
  - 每条命令的原始片段、修复结果、修复说明、复制按钮、风险提示。
  - 结果命令框可点击复制。
  - 如果未来开启显式多命令模式，再展示复制全部。
- 状态：
  - 空状态。
  - 成功状态。
  - 无命令状态。
  - here-doc 不支持状态。
  - 高风险提示状态。
  - 复制成功反馈。
- 响应式：
  - 桌面端采用上输入、下结果的工作台布局，给长命令更宽的核对空间。
  - 移动端单列，按钮不换行挤压。
  - 命令内容使用 monospace，并允许横向滚动，避免视觉换行造成误解。

验收：

- 用户打开页面不需要说明即可粘贴、修复、复制。
- 单条长命令能复制按钮复制，也能点击结果框复制。
- 显式多命令模式后续再验收分别复制与复制全部。
- 刷新页面后不恢复上一条命令内容。
- 移动端可完成粘贴、修复、复制，不出现按钮遮挡。

### 3.3 本地偏好存储

负责人：前端 Agent。

文件：`src/storage.js`

实现内容：

- 保存偏好：
  - 是否显示差异。
  - 是否显示原始片段。
  - 自定义命令前缀。
- 不保存：
  - 输入全文。
  - 修复结果。
  - 命令历史。
- 提供清除本地设置。
- 处理 `localStorage` 不可用的降级场景。

验收：

- 刷新后偏好保留。
- 刷新后输入内容和命令结果不保留。
- 清除本地设置后偏好恢复默认。

## 6. 风险与应对

- 风险：默认单命令模式会把命令后的说明文字并入结果。
  - 应对：MVP 默认用户知道命令范围；后续通过显式“多命令/智能识别模式”解决。
- 风险：多命令识别误拆复杂 shell 结构。
  - 应对：MVP 默认关闭自动拆分；auto 模式保留 shell block 保护。
- 风险：状态机修复破坏引号内空格。
  - 应对：测试覆盖 Python `-c`、SQL、JSON 参数。
- 风险：用户误以为工具会保证命令安全。
  - 应对：风险提示写明只提示、不保证。
- 风险：本地存储误保存敏感命令。
  - 应对：存储模块只暴露偏好 API，不提供历史保存 API。
- 风险：视觉做成营销页，影响工具效率。
  - 应对：首屏直接工具化，避免落地页结构。
