## 手工验收

启动本地静态服务：

```bash
npm start
```

然后打开 `http://localhost:4173`。

1. 点击“加载示例”，确认输出多条命令。
2. 逐条点击“复制”，确认剪贴板内容是单行命令。
3. 点击“复制全部”，确认多条可支持命令使用换行分隔，here-doc 不支持项会被跳过。
4. 修改“显示原始片段”“显示修复摘要”或自定义前缀，刷新页面后确认偏好保留。
5. 刷新页面后确认输入框和结果区不会恢复之前的命令内容。
6. 断网后重新打开已加载页面，确认修复与复制核心流程仍可使用。

说明：项目使用 ES module。通过本地静态服务访问比直接用 `file://` 打开 `index.html` 更接近真实部署环境，也能避免部分浏览器对本地模块加载的限制。
