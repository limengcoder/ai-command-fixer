## 手工验收

启动本地静态服务：

```bash
npm start
```

然后打开 `http://127.0.0.1:4173`。

1. 桌面端确认第一屏是左右分列：左侧输入、右侧结果；顶部和输入标题区没有设置按钮。
2. 确认左侧底部只有“修复命令”是主按钮，“试用示例”和“清空”是轻量次级操作。
3. 初始状态下确认右侧显示输出空状态，且不显示“复制全部”。
4. 点击“试用示例”，确认输出多条命令，此时才显示“复制全部”。
5. 输入含日期或文件名折行的命令，确认结果卡片显示“自动修复点”高亮。
6. 逐条点击“复制”，确认剪贴板内容是单行命令。
7. 点击“复制全部”，确认多条可支持命令使用换行分隔，here-doc 不支持项会被跳过。
8. 下滑到工作区下方，展开“设置”，修改“显示原始片段”“显示修复摘要”或自定义前缀，刷新页面后确认偏好保留。
9. 清除本地偏好后确认默认不显示原始片段和修复摘要。
10. 刷新页面后确认输入框和结果区不会恢复之前的命令内容。
11. 移动端窄屏确认页面变为单列，输入、结果、设置都可正常操作。
12. 断网后重新打开已加载页面，确认修复与复制核心流程仍可使用。

说明：项目使用 ES module。通过本地静态服务访问比直接用 `file://` 打开 `index.html` 更接近真实部署环境，也能避免部分浏览器对本地模块加载的限制。
